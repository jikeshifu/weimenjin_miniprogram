# 人脸过期编辑修复说明

## 问题

人脸过期后，用户通过编辑功能修改过期时间，数据库时间已更新，但设备仍提示“人脸过期”。

## 原因

原编辑逻辑只更新数据库中的 `end_time` 和 `face_name`，没有同步更新设备端的人脸信息。硬件云人脸接口没有直接编辑方法，只提供添加、特征值添加、删除和清空能力。

## 处理方式

编辑人脸时采用“先删除旧人脸，再重新添加”的方式同步设备：

1. 查询原有人脸和设备信息。
2. 删除设备上的旧人脸。
3. 优先使用 `face_feature` 重新添加人脸。
4. 如果没有特征值，则使用原图片 URL 重新添加。
5. 更新数据库中的人脸名称和过期时间。
6. 记录日志，便于排查设备同步异常。

## 涉及文件

- `app/api/controller/device/Face.php`
- `app/module/hardwareCloud/deivce/face.php`
- `weimenjin_app/pages/addFace/addFace.vue`
- `weimenjin_app/pages/faceList/faceList.vue`

## 验证建议

1. 选择一个已过期的人脸。
2. 修改过期时间为未来时间并提交。
3. 检查数据库 `cd_face` 中的 `end_time` 是否更新。
4. 使用该人脸在设备上开门，确认不再提示过期。
5. 查看 `face_edit.txt` 日志，确认同步过程无异常。

## 注意事项

- 设备需要在线，才能同步更新设备端人脸。
- 删除失败但重新添加成功时，最终结果通常仍然正确。
- 旧数据如果没有 `face_feature`，会走图片 URL 重新识别流程，速度会稍慢。
- 前端应避免重复提交，防止短时间内重复添加。
